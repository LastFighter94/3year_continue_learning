'/**': { isr: true } - что это значит?

Запись '/\*\*': { isr: true } в routeRules Nuxt 3 означает, что для всех маршрутов (/** — это шаблон, который соответствует всем путям) включён режим Incremental Static Regeneration (ISR).

Что такое ISR?
Incremental Static Regeneration (ISR) — это гибридный режим рендеринга, при котором страницы:

Генерируются статически на сервере при первом запросе (on-demand).

Кэшируются на CDN (например, Vercel или Netlify).

При последующих запросах пользователи получают кэшированную версию страницы.

В фоне происходит фоновая регенерация страницы (обновление кэша), чтобы контент оставался свежим без необходимости полного ребилда сайта.

Что значит isr: true?
Страница будет сгенерирована один раз при первом запросе и затем кэшироваться постоянно (до следующего деплоя).

Обновление страницы происходит только при деплое.

Пользователь всегда получает кэшированную версию, без задержек на рендеринг.

Если указать число, например, isr: 60, то:

Страница будет регенерироваться в фоне не чаще, чем раз в 60 секунд.

Это позволяет иметь свежий контент с минимальными задержками.
